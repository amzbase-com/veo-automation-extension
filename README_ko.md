## 🚀 왜 VEO Automation인가

Google Flow는 강력하지만, 공식 UI에서는 **프롬프트를 한 번에 하나씩** 실행하고 파일도 **하나씩** 다운로드해야 합니다. 숏폼 영상, 마케팅 크리에이티브, 스토리보드, 제품 컷처럼 콘텐츠를 대량으로 만든다면 이런 수작업 반복이 바로 병목입니다.

VEO Automation은 그 병목을 없앱니다.

- **목록을 붙여넣고 자리를 떠나세요.** 수십, 수백 개의 프롬프트를 넣어 두면 확장 프로그램이 재시도를 포함해 순차적으로 처리하는 동안 당신은 다른 일을 하면 됩니다.
- **VEO 3 영상*과* Nano Banana 이미지를 한 도구에서.** 대부분의 자동화 도구는 영상만 다룹니다. VEO Automation은 Google Flow의 영상 모델과 Nano Banana 이미지 모델을 하나의 사이드 패널에서 모두 배치 처리합니다.
- **파일이 알아서 디스크에 저장됩니다.** 완성된 모든 영상과 이미지는 이름이 정리되어 프로젝트별 폴더에 저장됩니다. 우클릭-저장을 반복할 필요가 없습니다.

*Google Flow 배치 도구*, *VEO 3 대량 생성기*, 또는 *Nano Banana 이미지 자동 다운로드* 방법을 찾고 있었다면, 바로 그 용도로 만든 도구입니다.

| 📝 **텍스트→영상 (VEO 3)** | 프롬프트를 붙여넣거나 수백 줄짜리 `.txt` 파일을 업로드해 영상을 대량 생산합니다. |

| 🎨 **텍스트→이미지 (Nano Banana)** | 텍스트 설명으로 이미지를 대량 생성합니다. |

| ⚙️ **세밀한 커스터마이징** | 모델, 화면 비율(16:9 / 9:16 / 1:1), 프롬프트당 결과 수(1–4), 동시 실행 수, 프롬프트 간 대기 시간을 직접 설정합니다. |
| 🌍 **7개 언어** | English, 中文, 한국어, 日本語, Deutsch, Français, Русский. |

## 📥 설치

### Chrome 웹 스토어 (권장)

1. Chrome 웹 스토어에서 [**VEO Automation 등록 페이지**]({{EXTENSION_URL}}?hl=ko)를 엽니다.
2. **Chrome에 추가** → **확장 프로그램 추가**를 클릭합니다.
3. 확장 프로그램을 고정하세요: 크롬 툴바의 퍼즐 조각 아이콘을 클릭한 뒤 **VEO Automation** 옆의 핀을 눌러 두면, 다음부터 한 번의 클릭으로 열 수 있습니다.

> **호환성:** Chrome 137 이상 (Edge, Brave 등 크로미움 기반 브라우저 포함). 확장 프로그램은 Google Flow 옆에 **사이드 패널**로 열립니다.

## ⚡ 빠른 시작

1. **Google Flow 열기.** [labs.google/fx/tools/flow](https://labs.google/fx/tools/flow)로 이동해 프로젝트를 열거나 만듭니다. 확장 프로그램은 Flow 프로젝트 페이지에서만 활성화됩니다.
2. **사이드 패널 열기.** VEO Automation 아이콘을 클릭합니다. 로그인하면 일일 할당량이 활성화됩니다.
3. **모드 선택.** 아래의 다섯 가지 생성 모드 중 하나를 고릅니다.
4. **프롬프트 추가.** 직접 입력하거나 `.txt` 파일을 업로드합니다.
5. **실행 클릭.** 대기열이 각 작업을 처리하고, 오류 시 재시도하며, 결과를 자동 다운로드하는 과정을 지켜보세요.

> ℹ️ **"디버깅 중" 배너에 대한 안내.** 확장 프로그램이 실행되는 동안 크롬 상단에 *"… 이(가) 디버깅되고 있습니다"* 라는 막대가 표시됩니다. 이는 꼭 필요한 과정입니다. Google Flow는 진짜로 신뢰할 수 있는 사용자 입력만 받아들이기 때문에, 확장 프로그램은 크롬의 공식 디버깅 인터페이스를 통해 페이지를 조작합니다. 악성 프로그램이 아니라, 안정적인 자동화를 위한 대가입니다. 배치가 실행되는 동안에는 이 배너를 닫지 마세요.

## 🎛️ 생성 모드

### 1. 텍스트→영상 — VEO 3

텍스트만으로 영상을 대량 생산합니다.

1. **텍스트→영상**을 선택합니다.
2. 입력 칸에 프롬프트를 입력하거나(각 프롬프트는 **빈 줄**로 구분) **또는** **.txt 업로드**를 클릭해 프롬프트 목록을 가져옵니다.
3. 길이를 설정합니다(단일 8초 클립, 또는 더 긴 이어붙이기 영상을 위한 **Concat**).
4. **실행**을 클릭합니다.

**프롬프트 목록 예시:**

```
황금빛 노을이 지는 절벽 위의 외딴 등대, 아래로 부서지는 파도, 느리고 영화적인 돌리 인.

비 내리는 네온 불빛의 도쿄 골목, 젖은 아스팔트에 비친 반사광, 피어오르는 수증기, 핸드헬드 느낌.

가을 숲 위를 미끄러지듯 나는 항공 촬영, 나무 사이로 낮게 깔린 아침 안개, 따뜻한 빛.
```

### 2. 이미지→영상 — VEO 3

정지 프레임에 생명을 불어넣습니다.

1. **이미지→영상**을 선택합니다.
2. **이미지 선택**을 클릭해(다중 선택 가능) 이름, 날짜, 사용자 지정 순서로 끌어서 재정렬합니다.
3. 이미지마다 프롬프트를 작성합니다(빈 줄로 구분).
4. 길이를 고른 뒤 **실행**합니다.

> **팁:** 시작 프레임 + 끝 프레임을 사용하면 움직임이 시작되고 끝나는 방식을 제어할 수 있습니다. 설정에서 프롬프트당 이미지 1–2장을 지정하세요.

### 3. 텍스트→이미지 — Nano Banana

설명만으로 이미지를 대량 생성합니다.

1. **텍스트→이미지**를 선택합니다.
2. 상세한 이미지 프롬프트를 입력합니다(빈 줄로 구분).
3. 설정에서 화면 비율과 이미지 품질을 지정합니다.
4. **실행** — 모든 이미지가 선택한 해상도(1K / 2K / 4K)로 자동 다운로드됩니다.

### 4. 이미지→이미지 — Nano Banana

참조 이미지를 대규모로 편집하거나 스타일을 바꿉니다.

1. **이미지→이미지**를 선택합니다.
2. 원본 이미지를 업로드합니다.
3. 변형 프롬프트를 작성합니다(예: *"같은 캐릭터, 사이버펑크 복장, 밤의 옥상"*).
4. **실행**하면 참조 이미지를 기반으로 변형 결과가 생성됩니다.

> **파일명으로 캐릭터 자동 매칭 (선택):** 참조 이미지를 `dog.png`, `cat.png`, `duck.png`처럼 이름 붙이세요. *"개가 고양이와 노는"* 내용을 언급한 프롬프트는 `dog.png`와 `cat.png`를 자동으로 가져옵니다. 시리즈 전체에서 캐릭터를 일관되게 유지하기에 안성맞춤입니다.

### 5. 대기열 관리

- **프롬프트 대기열**에서 대기 중인 모든 작업을 확인합니다.
- 더 이상 필요 없는 작업은 **삭제**하거나, 실행 중인 배치는 **중지**할 수 있습니다.
- 프롬프트별 진행 상황을 실시간으로 추적합니다.

## ⚙️ 설정 안내

**설정** 탭을 열어 동작을 세밀하게 조정하세요.

**일반**
- **기본 모드** — 패널을 열 때마다 자동으로 선택됩니다.
- **기본 화면 비율** — 16:9, 9:16, 또는 1:1.
- **프롬프트당 결과 수** — 프롬프트당 영상/이미지 1–4개.
- **동시 실행 프롬프트** — 동시에 1–6개의 프롬프트를 실행합니다.
- **프롬프트 대기 시간** — 서버 부하를 줄이기 위해 프롬프트 사이에 0–300초 대기합니다.

**모델**
- **영상 모델** — VEO 모델 선택(예: VEO 3 / VEO 3 Fast).
- **이미지 모델** — 텍스트→이미지 및 이미지→이미지용 Nano Banana.

**다운로드**
- **영상 품질** — 720p, 1080p, 또는 끄기.
- **이미지 품질** — 1K, 2K, 4K, 또는 끄기.
- 파일은 크롬 다운로드 폴더에 프로젝트별 하위 폴더로 저장됩니다.

**고급**
- **최대 재시도 횟수** — 실패 시 1–20회 시도.
- **기본 영상 프레임** — 8초 또는 Concat.
- **프롬프트당 최대 이미지 수** — 모드별로 설정.
- **언어** — 7개 언어 중에서 UI 언어를 전환합니다.

## 💡 팁 & 모범 사례

**프롬프트 작성**
- 주제, 스타일, 카메라 움직임, 조명을 구체적으로 적으세요.
- Google Flow에서는 영어 프롬프트가 가장 안정적인 경향이 있습니다.
- 프롬프트는 한 줄의 빈 줄로 구분하세요.

**처리량 vs. 안정성**
- 비혼잡 시간대: 동시 실행 약 3개가 잘 작동합니다.
- 혼잡 시간대: 동시 실행을 2개로 줄이고 대기 시간을 약 30초로 높이면 "생성 실패" 오류가 줄어듭니다.
- 동시 실행이 많을수록 빠르지만 덜 안정적입니다. 취향에 맞게 조정하세요.

**에셋 관리**
- 자동 매칭이 작동하도록 참조 이미지 이름을 명확하게 지으세요.
- PNG / JPG / GIF를 사용하고, 가급적 파일당 10MB 이하로 유지하세요.

**안정성**
- 밤새 자리를 비우고 돌리는 배치에는 **자동 재시도**를 켜 두세요.
- 대기 시간이 길수록 서버 부하가 가벼워지고 실패가 줄어듭니다.

## 🔧 문제 해결

| 증상 | 원인 & 해결 |
| --- | --- |
| **확장 프로그램이 아무 반응이 없음** | 반드시 Google Flow **프로젝트** 페이지에 있어야 합니다. 확장 프로그램이 켜져 있는지 확인하고, 새로고침한 뒤 패널을 다시 여세요. |
| **"영상을 만들 수 없음"** | Google Flow가 일시적으로 과부하 상태입니다. 걱정 마세요. 확장 프로그램이 자리가 빌 때까지 약 30초마다 기다렸다가 재시도합니다. |
| **영상/이미지가 다운로드되지 않음** | 크롬에서 **설정 → 다운로드 → "다운로드하기 전에 각 파일의 저장 위치 확인" 끄기**. 그런 다음 설정에서 다운로드 품질을 다시 확인하세요. |
| **"정책 오류"** | 프롬프트나 이미지가 구글의 콘텐츠 정책에 걸렸습니다. 도구가 해당 작업을 건너뛰고 자동으로 다음으로 넘어갑니다. |
| **생성이 계속 실패함** | 연결 상태를 확인하고, 프롬프트가 유효한지 점검하고, 최대 재시도 횟수를 높인 뒤 브라우저 콘솔에서 단서를 살펴보세요. |
| **페이지가 저절로 축소됨** | 의도된 동작입니다. 도구가 버튼을 정확히 찾기 위해 확대/축소를 조정합니다. 배치 실행 중에는 직접 확대/축소를 바꾸지 마세요. |

## ❓ 자주 묻는 질문(FAQ)

**VEO Automation은 Google과 제휴 관계인가요?**
아니요. 공개된 Google Flow 웹 앱을 자동화하는 독립적인 서드파티 도구입니다. Google과 제휴, 보증, 또는 어떠한 연관 관계도 없습니다.

**Nano Banana가 무엇인가요?**
Nano Banana는 Google Flow에서 사용할 수 있는 이미지 모델입니다. VEO Automation을 사용하면 VEO 3 영상과 함께 Nano Banana 이미지를 같은 대기열에서 배치로 생성하고 자동 다운로드할 수 있습니다.

**정말 한 번에 수백 개의 영상을 생성할 수 있나요?**
네. 프롬프트를 담은 `.txt` 파일을 업로드하거나(또는 긴 목록을 붙여넣으면) 대기열이 자동 재시도와 함께 하나씩 처리합니다. 실제 처리량은 Google Flow의 부하와 동시 실행/대기 시간 설정에 따라 달라집니다.

**숏폼 세로 영상에도 쓸 수 있나요?**
네 — 화면 비율을 TikTok / Reels / Shorts용 **9:16**, YouTube용 **16:9**, 정사각형용 **1:1**로 설정하세요.

**파일은 어디에 저장되나요?**
크롬 다운로드 폴더에 프로젝트별 하위 폴더로 정리되며, 정렬하기 쉽도록 자동으로 이름이 바뀝니다.

**API 키가 필요한가요?**
아니요. 브라우저에서 직접 로그인한 본인의 Google Flow 세션을 조작하므로, 별도의 API 키나 개발자 계정이 필요 없습니다.

**제 데이터가 어딘가로 전송되나요?**
프롬프트, 이미지, 영상은 브라우저 내에서 Google Flow를 통해 처리됩니다. 확장 프로그램은 설정을 로컬에 저장합니다. [개인정보 & 권한](#-개인정보--권한)을 참고하세요.

**어떤 브라우저를 지원하나요?**
Chrome 137 이상 및 크로미움 기반 브라우저(Edge, Brave). 사이드 패널이 필요합니다.
