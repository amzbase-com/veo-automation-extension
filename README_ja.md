## 🚀 なぜ VEO Automation なのか

Google Flow は強力ですが、公式 UI では **プロンプトを1件ずつ** 実行し、ファイルを **1つずつ** ダウンロードするしかありません。短尺動画、マーケティングクリエイティブ、絵コンテ、商品ショットなど、コンテンツを大量に作る人にとって、この手作業のループこそがボトルネックです。

VEO Automation は、それを取り除きます:

- **リストを貼って、あとは放置。** 数十〜数百のプロンプトを投入すれば、拡張機能が再試行を挟みながら順番に処理。その間、あなたは別の作業に集中できます。
- **VEO 3 動画 *と* Nano Banana 画像を1つのツールで。** 多くの自動化ツールは動画にしか対応していません。本拡張機能は、Google Flow の動画モデルと Nano Banana 画像モデルの両方を、1つのサイドパネルからバッチ処理します。
- **ファイルは自動でディスクへ。** 完成した動画・画像はすべてリネームされ、プロジェクトごとのフォルダに保存。右クリック保存のマラソンはもう不要です。

*Google Flow バッチツール*、*VEO 3 一括生成ツール*、あるいは *Nano Banana 画像を自動ダウンロードする方法* を探してここにたどり着いたなら、まさにそのために作られたツールです。

| 機能 | 内容 |
| --- | --- |
| 🚀 **バッチキュー** | 無制限のプロンプトを待機リストに追加。タスクは自動で次々と実行されます。 |
| 📝 **テキストから動画（VEO 3）** | プロンプトを貼り付けるか、数百行の `.txt` ファイルをアップロードして動画を大量生成。 |
| 🖼️ **画像から動画（VEO 3）** | 開始フレーム／終了フレーム制御で、静止画をアニメーション化。 |

| 🔄 **画像から画像（Nano Banana）** | 参照画像を大規模に変換・編集。 |
| 💾 **自動ダウンロード** | 完成した動画・画像を即座にローカルへ保存。プロジェクトフォルダごとに整理されます。 |

## 📥 インストール

### Chrome ウェブストア（推奨）

1. Chrome ウェブストアで [**VEO Automation の掲載ページ**](https://chromewebstore.google.com/detail/veo-flow-automation-%E2%80%94-bat/inmkimakahiennggieafocbmfjlmfbhn?hl=ja) を開きます。
2. **Chrome に追加** → **拡張機能を追加** をクリック。
3. 拡張機能をピン留め: Chrome ツールバーのパズルピースアイコンをクリックし、**VEO Automation** の横のピンをクリック。次回からワンクリックで起動できます。

> **対応環境:** Chrome 137 以降（Edge や Brave などの Chromium ベースのブラウザを含む）。拡張機能は Google Flow の横に **サイドパネル** として開きます。

## ⚡ クイックスタート

1. **Google Flow を開く。** [labs.google/fx/tools/flow](https://labs.google/fx/tools/flow) にアクセスし、プロジェクトを開く（または作成）します。拡張機能は Flow のプロジェクトページでのみ有効になります。
2. **サイドパネルを開く。** VEO Automation のアイコンをクリック。サインインすると、1日の無料枠が解放されます。
3. **モードを選ぶ。** 5つの生成モード（後述）から1つを選択します。
4. **プロンプトを追加する。** 直接入力するか、`.txt` ファイルをアップロードします。
5. **実行をクリック。** キューが各タスクを処理し、エラー時には再試行し、成果物を自動ダウンロードする様子を見守りましょう。

> ℹ️ **「デバッグ中」バナーについて。** 拡張機能の実行中、Chrome の上部に *「… をデバッグしています」* というバーが表示されます。これは必要な動作です: Google Flow は本物の信頼できるユーザー入力のみを受け付けるため、拡張機能は Chrome 公式のデバッグインターフェースを通じてページを操作します。マルウェアではありません — 信頼性の高い自動化のために必要なコストです。バッチの実行中はバナーを閉じないでください。

## 🎛️ 生成モード

### 1. テキストから動画 — VEO 3

テキストから直接、動画を大量生産します。

1. **テキストから動画** を選択。
2. ボックスにプロンプトを入力（各プロンプトは **空行** で区切る）するか、**.txt をアップロード** をクリックしてプロンプトリストを取り込みます。
3. 長さを設定（8秒の単一クリップ、またはより長い連結動画は **Concat**）。
4. **実行** をクリック。

**プロンプトリストの例:**

```
ゴールデンアワーの崖に立つ孤独な灯台、下では波が砕け、ゆっくりとシネマティックにドリーイン。

雨に濡れたネオン輝く東京の路地、濡れたアスファルトに映る反射、立ちのぼる湯気、手持ちカメラの質感。

秋の森を滑空する空撮、木々の間に立ちこめる朝靄、暖かな光。
```

### 2. 画像から動画 — VEO 3

静止フレームに命を吹き込みます。

1. **画像から動画** を選択。
2. **画像を選択**（複数選択対応）をクリックし、名前・日付・任意の順序でドラッグして並べ替えます。
3. 画像ごとにプロンプトを記述（空行で区切る）。
4. 長さを選んで **実行**。

> **ヒント:** 開始フレーム + 終了フレームを使えば、モーションの始まりと終わりを制御できます。設定でプロンプトあたりの画像数を 1〜2 枚に指定してください。

### 3. テキストから画像 — Nano Banana

説明文から画像を、まとめて生成します。

1. **テキストから画像** を選択。
2. 詳細な画像プロンプトを入力（空行で区切る）。
3. 設定でアスペクト比と画質を指定。
4. **実行** — すべての画像が、選んだ解像度（1K / 2K / 4K）で自動ダウンロードされます。

### 4. 画像から画像 — Nano Banana

参照画像を大規模に編集・スタイル変更します。

1. **画像から画像** を選択。
2. ソース画像をアップロード。
3. 変換用のプロンプトを記述（例: *「同じキャラクター、サイバーパンクの衣装、夜の屋上」*）。
4. **実行** すると、参照画像をもとにバリエーションを生成します。

> **ファイル名でキャラクターを自動マッチング（任意）:** 参照画像を `dog.png`、`cat.png`、`duck.png` のように命名します。*「dog playing with cat（犬が猫と遊ぶ）」* と書かれたプロンプトは、自動的に `dog.png` と `cat.png` を取り込みます。シリーズ全体でキャラクターの一貫性を保つのに最適です。

### 5. キュー管理

- **プロンプトキュー** で、待機中のすべてのタスクを確認。
- 不要になったタスクを **削除** したり、実行中のバッチを **停止** したり。
- プロンプトごとの進捗をリアルタイムで追跡。

## ⚙️ 設定リファレンス

**設定** タブを開いて、動作を調整できます。

**一般**
- **デフォルトモード** — パネルを開くたびに自動選択されます。
- **デフォルトのアスペクト比** — 16:9、9:16、または 1:1。
- **プロンプトあたりの出力数** — プロンプトごとに 1〜4 本の動画／画像。
- **同時実行プロンプト数** — 1〜6 件のプロンプトを同時に実行。
- **プロンプトディレイ** — サーバー負荷を抑えるため、プロンプト間で 0〜300 秒待機。

**モデル**
- **動画モデル** — VEO モデルを選択（例: VEO 3 / VEO 3 Fast）。
- **画像モデル** — テキストから画像、画像から画像向けの Nano Banana。

**ダウンロード**
- **動画品質** — 720p、1080p、またはオフ。
- **画像品質** — 1K、2K、4K、またはオフ。
- ファイルは Chrome のダウンロードフォルダに、プロジェクトごとのサブフォルダで保存されます。

**詳細設定**
- **最大リトライ回数** — 失敗時に 1〜20 回まで再試行。
- **デフォルトの動画フレーム** — 8秒 または Concat。
- **プロンプトあたりの最大画像数** — モードごとに設定。
- **言語** — UI を 7言語の中から切り替え。

## 💡 ヒントとベストプラクティス

**プロンプトの書き方**
- 被写体、スタイル、カメラの動き、ライティングを具体的に。
- Google Flow では、英語のプロンプトが最も安定する傾向があります。
- プロンプト同士は1つの空行で区切ってください。

**スループット vs. 安定性**
- オフピーク時: 同時実行 ~3 件がうまく機能します。
- ピーク時: 2 件に減らし、ディレイを ~30 秒に上げると「作成失敗」エラーが減ります。
- 同時実行数を増やすほど速くなりますが、安定性は下がります。好みに合わせて調整を。

**アセットの管理**
- 自動マッチングが機能するよう、参照画像には分かりやすい名前を付けましょう。
- 形式は PNG / JPG / GIF に統一し、できれば各 10 MB 未満に。

**信頼性**
- 夜間の無人バッチでは **自動リトライ** をオンのままに。
- ディレイを長くするほどサーバー負荷が軽くなり、失敗が減ります。

## 🔧 トラブルシューティング

| 症状 | 原因と対処 |
| --- | --- |
| **拡張機能が何も反応しない** | Google Flow の **プロジェクト** ページにいる必要があります。拡張機能が有効か確認し、ページを更新してパネルを開き直してください。 |
| **「動画を作成できません」** | Google Flow が一時的に混雑しています。ご心配なく — 拡張機能は空きが出るまで ~30 秒ごとに待機・再試行します。 |
| **動画／画像がダウンロードされない** | Chrome で: **設定 → ダウンロード → 「ダウンロード前に各ファイルの保存場所を確認する」をオフ** にします。その後、設定でダウンロード品質を再確認してください。 |
| **「ポリシーエラー」** | プロンプトまたは画像が Google のコンテンツポリシーに抵触しました。ツールはそのタスクをスキップし、自動的に次へ進みます。 |
| **生成が失敗し続ける** | 接続を確認し、プロンプトが有効かをチェックし、最大リトライ回数を上げ、ブラウザのコンソールで手がかりを確認してください。 |
| **ページが勝手に縮小した** | 意図的な動作です — ツールはボタンを正確に特定するためにズームを調整します。バッチ実行中は手動でズームを変更しないでください。 |
| **「デバッグ中」バナーが消えない** | 実行中は想定どおりの動作です（クイックスタート参照）。バッチが完了するか、停止すると消えます。 |

## ❓ よくある質問（FAQ）

**VEO Automation は Google と提携していますか？**
いいえ。これは、公開されている Google Flow ウェブアプリを自動化する、独立したサードパーティ製ツールです。Google との提携・承認・関連は一切ありません。

**Nano Banana とは何ですか？**
Nano Banana は、Google Flow を通じて利用できる画像モデルです。VEO Automation を使えば、VEO 3 動画とあわせて、同じキューから Nano Banana 画像をバッチ生成・自動ダウンロードできます。

**本当に一度に何百もの動画を生成できますか？**
はい。プロンプトの `.txt` ファイルをアップロード（または長いリストを貼り付け）すれば、キューが1件ずつ自動リトライ付きで処理します。実際のスループットは、Google Flow の負荷状況と、同時実行数／ディレイの設定によって変わります。

**短尺の縦型動画にも対応していますか？**
はい — TikTok / Reels / Shorts には **9:16**、YouTube には **16:9**、正方形には **1:1** にアスペクト比を設定してください。

**ファイルはどこに保存されますか？**
Chrome のダウンロードフォルダに、プロジェクトごとのサブフォルダで整理され、並べ替えやすいよう自動的にリネームされて保存されます。

**API キーは必要ですか？**
いいえ。ブラウザ上で、あなた自身がログイン済みの Google Flow セッションを操作します — 別途の API キーや開発者アカウントは不要です。

**自分のデータはどこかに送信されますか？**
プロンプト、画像、動画は、ブラウザ内の Google Flow 上で処理されます。拡張機能は設定をローカルに保存します。[プライバシーと権限](#-プライバシーと権限) をご覧ください。

**対応ブラウザは？**
Chrome 137 以降と、Chromium ベースのブラウザ（Edge、Brave）。サイドパネル機能が必要です。

## 🔒 プライバシーと権限

- **ブラウザ内で動作。** 自動化は、あなた自身の Google Flow セッション上で行われます。
- **最小限のデータ。** 設定は Chrome のローカルストレージに保存されます。拡張機能があなたの創作コンテンツを販売・収集することはありません。
- **限定された権限。** ホストアクセスは `labs.google/*`（Flow を自動化するため）と、拡張機能自身のバックエンド（サインインと利用枠のため）に限定されています。
- **デバッガー権限** は、ページが要求する信頼された入力イベントを Google Flow に送るためだけに使用されます — [クイックスタート](#-クイックスタート) を参照。

詳細はこちら: [https://github.com/amzbase-com/veo-automation-extension](https://github.com/amzbase-com/veo-automation-extension)
